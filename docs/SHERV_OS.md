# Sherv OS Integration

HeadRoom participates in Sherv OS as the AI capacity, quota, and cost-intelligence module.

- ChatGPT: executive brain and final decision layer.
- Cursor: primary supervised editor.
- Codex: scoped background implementation.
- Linear: operational truth.
- GitHub: technical truth.

## Controls

1. Read the repository and linked Linear issue before editing.
2. Keep provider integrations read-only unless a separately approved write action exists.
3. Never log or expose API keys, tokens, account identifiers, raw billing exports, or private usage data.
4. Prefer deterministic collection and arithmetic; use AI for interpretation and recommendations.
5. Run native lint, typecheck, tests, build, and provider-adapter tests.
6. Production deploys, credentials, billing settings, destructive data changes, and external publishing require human approval.
7. Every release documents validation, remaining risks, rollback, and recording artifacts.
8. Missing provider access must fail closed and appear clearly in capability health.

HeadRoom is the cost and capacity watchdog for the wider Sherv OS ecosystem.
